## 钩子命名规范

| 前缀 | 含义 | 用途 |
|------|------|------|
| `before` | 阶段开始前 | 预处理、校验参数 |
| `on` | 阶段执行中 | 核心逻辑、可修改数据 |
| `after` | 阶段完成后 | 后处理、清理工作 |
| `emit` | 触发事件 | 通知、日志、不可修改 |
| `resolve` | 解析/决定 | 计算最终值 |

```

[INIT]

  ├── beforeOptions

          │
          ▼

  ├── beforeParse

  ├── beforeResolve
```

## 使用示例

```typescript
import { runPipeline } from "@addfox/cli/pipeline";
import type { PipelineHooks } from "@addfox/core";

const hooks: PipelineHooks = {
  // 在解析选项之前
  beforeOptions: async (ctx) => {
    console.log("开始解析选项...");
  },

  // 在加载配置之后
  afterLoad: async (ctx) => {
    console.log("配置已加载:", ctx.config);
  },

  // 在构建配置时
  onBuild: async (ctx) => {
    // 可以修改 Rsbuild 配置
    ctx.rsbuild.plugins = ctx.rsbuild.plugins || [];
    ctx.rsbuild.plugins.push(myPlugin);
  },

  // 监听警告
  emitWarning: (ctx, message, ...args) => {
    console.warn("[Warning]", message, ...args);
  },

  // 流程完成
  onComplete: async (ctx) => {
    console.log("Pipeline 完成!");
  },
};

const ctx = await runPipeline(root, argv, hooks);
```

## 与旧 LifecycleHooks 的映射

| 旧钩子 (LifecycleHooks) | 新钩子 (PipelineHooks) | 说明 |
|------------------------|------------------------|------|
| afterCliParsed | afterParse | CLI 解析完成后 |
| afterConfigLoaded | afterLoad | 配置加载完成后 |
| beforeRsbuildConfig | beforeBuild | 构建配置前 |
| beforeBuild | onBuild | 构建配置时 |
| afterBuild | afterBuild | 构建配置后 |
