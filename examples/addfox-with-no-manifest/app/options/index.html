Addfox - No Manifest Example

Experience the power of auto-generated manifests

⚡ Zero Config

No manifest.json needed. Just create entry folders and Addfox handles the rest.

🔍 Auto Discovery

Entries are automatically discovered from app/background, app/content, app/popup folders.

📋 Smart Fill

Required fields like name, version, manifest_version are auto-filled with sensible defaults.

🎯 Type Aware

Addfox detects MV2/MV3 and generates the correct field format for each browser.

// addfox.config.ts - No manifest needed! export default defineConfig({ // Manifest will be auto-generated from entries }); // Auto-generated manifest.json: { "manifest_version": 3, "name": "Extension", "version": "1.0.0", "background": { "service_worker": "background/index.js" }, "content_scripts": [{ "matches": ["<all_urls>"], "js": ["content/index.js"] }], "action": { "default_popup": "popup/index.html" } }

📁 Entry Structure

  • app/background/index.ts → Auto-added as background.service_worker
  • app/content/index.ts → Auto-added as content_scripts[0]
  • app/popup/index.html → Auto-added as action.default_popup
  • app/options/index.html → Auto-added as options_ui.page
