# error.md

Когда Addfox обнаруживает ошибки времени выполнения в браузерном расширении, создается файл `error.md` и сохраняется в директории `.addfox/` в корне проекта.

## 1. Основная структура

Сгенерированный `error.md` содержит следующие стандартные блоки:

### Сводка ошибки

Предоставляет базовые метаданные ошибки, включая:
- **Entry**: Точка входа, где произошла ошибка (например, `content`, `background`, `popup`).
- **Type**: Тип ошибки (например, `error`, `warning`).
- **Time**: Локальное время возникновения ошибки.
- **Message**: Содержимое исходного сообщения об ошибке.
- **Location**: Позиция в исходном коде или путь к скомпилированному ресурсу, где произошла ошибка.

### Контекст сборки

Предоставляет базовую информацию о технологическом стеке сборки, такую как:
- **Bundler**: Название инструмента сборки (обычно `rsbuild`).
- **Framework**: Используемый UI фреймворк.

### Трассировка стека

Предоставляет полный стек вызовов JavaScript ошибки. AI может быстро отследить конкретную строку кода, вызвавшую сбой, анализируя эту часть.

---

> **Примечание**: Этот файл хранит только информацию о последней зафиксированной ошибке. При перезапуске сервера разработки или возникновении новой ошибки старое содержимое будет очищено.
