# Отчет об анализе

Addfox предоставляет возможности отчета об анализе сборки на основе **Rsdoctor** для позиционирования размера пакета, структуры зависимостей и узких мест сборки.

## Сценарии применения

- Необычное увеличение размера артефактов сборки
- Заметное замедление сборки после определенного изменения
- Анализ chunk entry, дублирующихся зависимостей и доли статических ресурсов

## Способы включения

Командная строка:

```bash
addfox build --report
```

Или включение в конфигурации:

```ts
export default defineConfig({
  report: true,
});
```

## Что можно увидеть

- Ситуация разделения entry и chunk
- Распределение размера зависимостей и дублирующиеся зависимости
- Длительность сборки и время на каждом этапе

## Рекомендуемый процесс

1. Сначала запустить базовый отчет;
2. После крупных изменений снова сгенерировать отчет;
3. Сравнить изменения размера и времени, затем определить направление оптимизации.
