# 分析レポート

Addfox は **Rsdoctor** に基づいてビルド分析レポート機能を提供し、パッケージサイズ、依存関係構造、ビルドのボトルネックの特定に使用されます。

## 適用シナリオ

- ビルド成果物のサイズが異常に増大した
- ある変更後にビルドが明らかに遅くなった
- エントリーチャンク、依存関係の重複、静的リソースの比率を分析したい

## 有効化方法

コマンドライン：

```bash
addfox build --report
```

または設定で有効にします：

```ts
export default defineConfig({
  report: true,
});
```

## 確認できる内容

- エントリーとチャンクの分割状況
- 依存関係のサイズ分布と重複
- ビルド時間とフェーズの所要時間

## 推奨フロー

1. まずベースラインレポートを1回実行;
2. 大きな変更後に再度レポートを生成;
3. サイズと所要時間の変化を比較し、最適化の方向性を決定。
