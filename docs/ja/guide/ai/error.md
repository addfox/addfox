# error.md

Addfox がブラウザ拡張機能の実行時エラーを検出すると、`error.md` ファイルが生成され、プロジェクトルートの `.addfox/` ディレクトリに保存されます。

## 1. コア構造

生成された `error.md` には以下の標準ブロックが含まれます：

### エラーサマリー

エラーの基本メタデータを提供します：
- **Entry**: エラーが発生したエントリーポイント（例：`content`、`background`、`popup`）。
- **Type**: エラータイプ（例：`error`、`warning`）。
- **Time**: エラーが発生したローカル時刻。
- **Message**: 元のエラーメッセージの内容。
- **Location**: エラーが発生したソースコードの位置またはコンパイル後のリソースパス。

### ビルドコンテキスト

ビルド技術スタックの基本情報を提供します：
- **Bundler**: ビルドツール名（通常は `rsbuild`）。
- **Framework**: 現在使用されている UI フレームワーク。

### スタックトレース

完全な JavaScript エラーコールスタックを提供します。AI はこの部分を分析することで、クラッシュの原因となった具体的なコード行に素早く遡ることができます。

---

> **注意**：このファイルは最後にキャプチャされたエラー情報のみを保持します。開発サーバーを再起動するか新しいエラーが発生すると、古い内容はクリアされます。
