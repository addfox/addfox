# debug

`debug`는 개발 시 오류 모니터링 패널을 활성화하는 데 사용됩니다.

## 개요

- **타입**: `boolean`
- **기본값**: `false`
- **필수 여부**: 아니오

## 사용법

```ts
// addfox.config.ts
import { defineConfig } from "addfox";

export default defineConfig({
  debug: true,  // 오류 모니터링 활성화
});
```

## 기능

활성화되면 Addfox는 다음을 수행합니다:

1. **오류 캡처** — background, content, popup 등 각 Entry의 오류를 자동으로 캡처
2. **오류 패널** — 확장 페이지에 모니터링 패널 Entry 표시
3. **AI 문의** — 오류를 한 번에 복사하여 ChatGPT/Cursor에 문의 가능

## 사용 방식

### 설정으로 활성화

```ts
export default defineConfig({
  debug: true,
});
```

### CLI로 활성화

## 모니터링 패널

활성화되면 `chrome-extension://<id>/_addfox-monitor/`에 접속하여 확인할 수 있습니다:

- 각 Entry의 오류 목록
- 오류 스택 상세 정보
- 오류 정보 한 번에 복사
- AI(ChatGPT / Cursor)에 한 번에 문의

## 주의사항

- 개발 모드(`addfox dev`)에만 영향을 줍니다
- 프로덕션 빌드는 모니터링 코드를 자동으로 제거합니다
- 모니터링 패널은 확장 성능에 영향을 주지 않습니다
