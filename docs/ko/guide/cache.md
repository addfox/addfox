# 캐시

Addfox는 프로젝트 아래에 `.addfox/cache` 디렉토리를 생성하여 개발 모드 경험을 향상시킵니다.

## `.addfox/cache`에 무엇이 있는가

일반적인 내용은 다음과 같습니다:

- 브라우저 사용자 데이터 캐시 (확장 로드 상태 재사용용)
- 빌드 프로세스의 중간 캐시 (반복 컴파일 오버헤드 감소)

다른 플랫폼과 실행 모드에 따라 캐시 파일 구조가 약간 다를 수 있지만, 역할은 동일합니다: **반복 초기화 비용 감소**.

## 어떤 역할을 하는가

가장 직접적인 이점은 개발 모드에서의 브라우저 시작 경험입니다:

- 첫 번째 `addfox dev`: 브라우저 환경 및 확장 로드를 완전히 초기화해야 함
- 이후 `addfox dev`: 캐시를 재사용한 후 브라우저가 디버깅 가능한 상태에 더 빠르게 진입하고 수동 작업을 줄입니다.

예를 들어 언급한 시나리오: 첫 번째 시작 후 캐시가 있으면 두 번째 시작은 일반적으로 동일한 프로세스를 다시 수동으로 열 필요가 없습니다.

## 언제 캐시를 정리하는가

다음과 같은 경우 `.addfox/cache`를 삭제하고 다시 시도할 수 있습니다:

- 브라우저 구성 상태가 비정상적일 때
- 확장 로드 동작이 예상과 일치하지 않을 때
- "완전히 새로 시작"하여 문제를 조사하려는 경우

해당 디렉토리를 직접 삭제할 수 있으며, Addfox는 다음 실행 시 자동으로 다시 생성합니다.
