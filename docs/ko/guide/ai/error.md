# error.md

Addfox가 브라우저 확장에서 런타임 오류를 모니터링하면 `error.md` 파일을 생성하여 프로젝트 루트 디렉토리의 `.addfox/` 디렉토리에 저장합니다.

## 1. 핵심 구조

생성된 `error.md`는 다음 표준 블록을 포함합니다:

### 오류 요약

오류의 기본 메타데이터를 제공하며, 다음을 포함합니다:
- **Entry**: 오류가 발생한 Entry 지점 (예: `content`, `background`, `popup`).
- **Type**: 오류 유형 (예: `error`, `warning`).
- **Time**: 오류가 발생한 로컬 시간.
- **Message**: 원시 오류 메시지 내용.
- **Location**: 오류가 발생한 소스 코드 위치 또는 컴파일된 리소스 경로.

### 빌드 컨텍스트

빌드 기술 스택의 기본 정보를 제공하며, 다음을 포함합니다:
- **Bundler**: 빌드 도구 이름 (일반적으로 `rsbuild`).
- **Framework**: 현재 사용 중인 UI 프레임워크.

### 스택 추적

전체 JavaScript 오류 호출 스택을 제공합니다. AI는 이 부분을 분석하여 충돌을 일으킨 구체적인 코드 라인을 빠르게 추적할 수 있습니다.

---

> **참고**: 이 파일은 가장 최근에 캡처된 오류 정볼만 유지합니다. 개발 서버를 다시 시작하거나 새로운 오류가 발생하면 이전 내용이 지워집니다.
