# error.md

当 Addfox 监控到浏览器扩展中的运行时错误时，会生成 `error.md` 文件并存储在项目根目录下的 `.addfox/` 目录中。

## 1. 核心结构

生成的 `error.md` 包含以下标准区块：

### 错误摘要

提供错误的基本元数据，包含：
- **Entry**: 发生错误的入口点（如 `content`, `background`, `popup`）。
- **Type**: 错误类型（如 `error`, `warning`）。
- **Time**: 错误发生的本地时间点。
- **Message**: 原始错误消息内容。
- **Location**: 发生错误的源代码位置或编译后资源路径。

### 构建上下文

提供构建技术栈的基本信息，如：
- **Bundler**: 构建工具名称（通常为 `rsbuild`）。
- **Framework**: 当前使用的 UI 框架。

### 堆栈追踪

提供完整的 JavaScript 错误调用栈。AI 通过分析此部分可以快速回溯到引起崩溃的具体代码行。

---

> **注意**：该文件仅保留最近一次捕获到的错误信息。当开发服务器重启或新错误发生时，旧内容将被清空。
